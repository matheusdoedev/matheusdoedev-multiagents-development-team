## Description

This skill provides a workflow for effectively using color in UI design. It emphasizes color theory, accessibility compliance, contrast ratios, and creating cohesive color systems.

## Scope

- Personal skill for UI/UX designers
- Applicable to web and app design
- Can be used for design system creation and color selection

## Workflow Steps

### 1. Color Theory Fundamentals

- Understand color models (RGB, HSL, HSV)
- Apply color harmony principles (complementary, analogous, triadic)
- Use warm and cool colors appropriately
- Understand color psychology
- Create visual hierarchy with color
- Use saturation and lightness effectively
- Apply contrast principles

### 2. Accessibility and Contrast

- Ensure WCAG AA minimum contrast (4.5:1 for text)
- Verify WCAG AAA contrast (7:1) when possible
- Test color contrast with tools
- Avoid relying on color alone
- Consider color blindness
- Test with colorblind simulators
- Document color combinations

### 3. Creating a Color Palette

- Define primary colors
- Define secondary colors
- Create neutral palette
- Define accent colors
- Consider dark mode variations
- Document color usage
- Maintain consistency

### 4. Color Variables and Tokens

- Define CSS custom properties
- Create color naming system
- Document color meanings
- Use semantic naming
- Support multiple themes
- Implement dark mode
- Test color combinations

### 5. UI Application

- Apply color to elements consistently
- Use color for status indication
- Use color for feedback
- Create visual hierarchy
- Ensure readability
- Consider cultural color meanings
- Test on different devices

### 6. Branding Integration

- Apply brand colors
- Maintain brand consistency
- Create brand guidelines
- Document color usage
- Share with team
- Maintain flexibility
- Evolve systematically

### 7. Dark Mode

- Design dark color palette
- Ensure sufficient contrast
- Test readability
- Adjust saturation for dark mode
- Consider eye strain
- Test on OLED screens
- Provide user control

### 8. Testing and Iteration

- Test on different monitors
- Test accessibility
- Get user feedback
- Test in different lighting
- Test printability
- Iterate based on feedback
- Monitor accessibility issues

## Quality Criteria

- Color palette is cohesive
- All contrast ratios meet WCAG AA
- Colors have clear purpose
- Design works in dark mode
- Design is accessible
- Color system is documented
- Team can use colors consistently
- Design communicates intent

## Example Usage

When creating a dashboard:

1. Define primary brand color
2. Create supporting color palette
3. Ensure high contrast for data visualization
4. Apply colors to different data states
5. Create dark mode variant
6. Test accessibility
7. Document color usage
