## Description

This skill provides a workflow for designing and building responsive web applications. It emphasizes mobile-first approach, flexible layouts, and testing across devices.

## Scope

- Personal skill for web designers and frontend developers
- Applicable to any web project
- Can be used for design, development, and testing

## Workflow Steps

### 1. Mobile-First Design

- Design for mobile first
- Create mobile wireframes
- Design touch-friendly interfaces
- Minimize horizontal scrolling
- Optimize font sizes
- Ensure readable line lengths
- Consider mobile performance

### 2. Flexible Layouts

- Use relative units (%, em, rem)
- Implement flexible grids
- Use Flexbox for layouts
- Use CSS Grid for complex layouts
- Avoid fixed widths
- Create flexible containers
- Test at all breakpoints

### 3. Responsive Images

- Use relative image sizing
- Implement responsive images with srcset
- Use picture element for art direction
- Optimize image sizes
- Use appropriate formats
- Consider loading performance
- Test image display

### 4. Breakpoints and Media Queries

- Define breakpoints based on content
- Use mobile-first media queries
- Implement tablet breakpoints
- Implement desktop breakpoints
- Test all breakpoints
- Avoid too many breakpoints
- Document breakpoint strategy

### 5. Typography

- Use scalable font sizes
- Maintain readable line length
- Adjust line height for readability
- Use appropriate font sizes per breakpoint
- Consider mobile readability
- Implement font scaling
- Test typography readability

### 6. Touch and Interaction

- Design touch-friendly buttons
- Ensure sufficient touch targets (44x44px)
- Optimize hover states for desktop
- Consider touch gestures
- Test on actual devices
- Avoid hover-dependent content
- Implement appropriate feedback

### 7. Performance

- Optimize for mobile bandwidth
- Lazy load images
- Minimize CSS and JavaScript
- Use compression
- Optimize critical path
- Test performance on slow networks
- Monitor metrics

### 8. Testing

- Test on actual devices
- Use responsive design tools
- Test keyboard navigation
- Test accessibility
- Test at different breakpoints
- Test touch interaction
- Test performance

## Quality Criteria

- Design works on all devices
- Layouts are flexible and adaptable
- Performance is optimized
- Touch interactions work well
- Accessibility is maintained
- Testing is comprehensive
- Design is consistent
- Performance metrics meet standards

## Example Usage

When designing a product listing:

1. Design mobile layout first
2. Implement flexible grid system
3. Add tablet breakpoint
4. Add desktop breakpoint
5. Optimize images for different sizes
6. Test touch interaction
7. Verify accessibility
8. Test on actual devices
