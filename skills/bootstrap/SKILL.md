## Description

This skill provides a workflow for building responsive interfaces using Bootstrap framework. It focuses on leveraging pre-built components, proper grid usage, customization, and accessibility compliance.

## Scope

- Personal skill for frontend developers using Bootstrap
- Applicable to Bootstrap 5 and later versions
- Can be used for rapid UI development and component composition

## Workflow Steps

### 1. Bootstrap Setup and Configuration

- Include Bootstrap CSS and JavaScript properly
- Configure Bootstrap with custom variables
- Customize colors, typography, and spacing
- Understand Bootstrap's design tokens
- Set up build tools for compilation
- Configure dark mode support

### 2. Grid System

- Understand 12-column grid layout
- Use container, row, col classes appropriately
- Implement responsive grid with breakpoints
- Use offset and order utilities
- Use gutters for consistent spacing
- Create nested grids when needed

### 3. Component Usage

- Leverage pre-built components (buttons, cards, modals)
- Customize component appearance with utilities
- Combine components for complex layouts
- Use component variants appropriately
- Implement state variations (active, disabled, loading)
- Document component usage patterns

### 4. Styling and Customization

- Use Bootstrap variables for consistent theming
- Create custom CSS for app-specific styles
- Extend Bootstrap components when needed
- Use utility classes effectively
- Avoid overriding Bootstrap styles unless necessary
- Document customization approach

### 5. JavaScript Components

- Use Bootstrap JS components (modals, dropdowns, tooltips)
- Initialize components properly
- Handle component events
- Configure component options
- Clean up components on removal
- Test component functionality

### 6. Responsive Design

- Use Bootstrap breakpoints appropriately
- Test on all breakpoints
- Hide/show elements responsively
- Implement responsive typography
- Adjust spacing for different screen sizes
- Consider mobile-first approach

### 7. Accessibility

- Ensure color contrast meets standards
- Use proper ARIA attributes
- Implement keyboard navigation
- Test with screen readers
- Use semantic HTML with Bootstrap
- Document accessibility features

## Quality Criteria

- UI uses Bootstrap components appropriately
- Customization is consistent and documented
- Responsive design works on all devices
- Accessibility standards are met
- CSS is optimized and minimal
- Components are reusable
- Code follows Bootstrap conventions

## Example Usage

When building a dashboard:

1. Set up Bootstrap grid for layout
2. Use card components for data
3. Customize colors and typography
4. Implement responsive breakpoints
5. Add interactive components (modals, dropdowns)
6. Ensure accessibility compliance
7. Test across devices and browsers
