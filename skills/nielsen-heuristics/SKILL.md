## Description

This skill provides a workflow for evaluating interfaces against Nielsen's 10 Usability Heuristics for User Interface Design. It emphasizes user control, system feedback, error prevention, and overall usability.

## Scope

- Personal skill for UX designers, developers, and product managers
- Applicable to any interface design
- Can be used for evaluation, design, and improvement

## Workflow Steps

### 1. Visibility of System Status

- Keep users informed in real time
- Provide appropriate feedback
- Use clear language
- Show system state
- Indicate loading states
- Show progress for long operations
- Provide status updates

### 2. Match Between System and Real World

- Use user's language
- Follow real-world conventions
- Use familiar metaphors
- Provide recognizable patterns
- Minimize jargon
- Use clear terminology
- Match user expectations

### 3. User Control and Freedom

- Provide undo/redo
- Offer emergency exits
- Allow cancellation
- Support back navigation
- Don't force workflows
- Allow alternative paths
- Provide clear "way out"

### 4. Error Prevention

- Prevent problems
- Ask for confirmation
- Suggest constraints
- Validate input
- Provide clear instructions
- Prevent errors before they occur
- Use constraints wisely

### 5. Error Recovery

- Provide clear error messages
- Suggest solutions
- Use plain language
- Indicate error severity
- Highlight problem areas
- Make recovery easy
- Avoid technical jargon

### 6. Recognition vs Recall

- Make options visible
- Minimize memory load
- Use clear labels
- Provide defaults
- Suggest next steps
- Use consistent patterns
- Support recognition

### 7. Flexibility and Efficiency

- Provide shortcuts for experts
- Support customization
- Allow preferences
- Offer multiple paths
- Use keyboard shortcuts
- Provide advanced options
- Balance simplicity and power

### 8. Aesthetic and Minimalist Design

- Remove unnecessary elements
- Focus on essentials
- Use white space
- Simplify visual design
- Reduce cognitive load
- Maintain consistency
- Support scanning

### 9. Help and Documentation

- Provide help easily
- Use task-focused help
- Provide clear instructions
- List concrete steps
- Avoid large sections
- Make searchable
- Test help effectiveness

### 10. Error Diagnosis and Recovery

- Explain problems clearly
- Suggest solutions
- Use user's language
- Be constructive
- Help prevent recurrence
- Enable recovery
- Learn from errors

## Quality Criteria

- Interface follows most heuristics
- Users feel in control
- System status is clear
- Error messages help users
- Design is intuitive
- Feedback is immediate
- Documentation is helpful
- Usability is high

## Evaluation Framework

- Rating: 0 (not present) to 5 (optimal)
- Assess each heuristic
- Identify violations
- Prioritize improvements
- Test improvements
- Re-evaluate
- Document findings

## Example Usage

When evaluating a checkout flow:

1. Check system feedback (step progress)
2. Verify clear language
3. Test undo/recovery options
4. Evaluate error messages
5. Check help availability
6. Test efficiency for returning users
7. Verify minimalist design
8. Assess overall usability
